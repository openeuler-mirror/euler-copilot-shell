# **Witty Assistant 部署指南**

版本信息
当前版本：v0.10.2
更新日期：2025年9月10日

## 产品概述

Witty Assistant 是一款智能问答工具，使用 Witty Assistant 可以解决操作系统知识获取的便捷性，并且为 OS 领域模型赋能开发者及运维人员。作为获取操作系统知识，使能操作系统生产力工具 (如 A-Ops / A-Tune / x2openEuler / EulerMaker / EulerDevOps / StratoVirt / iSulad 等)，颠覆传统命令交付方式，由传统命令交付方式向自然语义进化，并结合智能体任务规划能力，降低开发、使用操作系统特性的门槛。

本指南提供基于自动化脚本的 Witty Assistant 智能助手部署说明，支持一键自动部署和手动分步部署两种方式。

### 组件介绍

| 组件                          | 端口            | 说明                  |
| ----------------------------- | --------------- | -------------------- |

| sysagent                       | 8002            | 智能体框架服务         |

## 快速开始

### 1. 获取轻量化部署客户端

```bash
dnf install witty-assistant-installer
```

### 2. 修改基础配置文件

```bash
[root@localhost ~]# cd /usr/lib/witty-assistant/scripts/5-resource/
[root@localhost 5-resource]# ll
总计 1700
-rwxr-xr-x. 1 root root  1681126 2025年 7月16日 9b5ad71b2ce5302211f9c61530b329a4922fc6a4
-rwxr-xr-x. 1 root root     1340  7月10日 15:40 config.toml
-rwxr-xr-x. 1 root root      868 2025年 7月16日 token.py

[root@localhost 5-resource]# vim config.toml  # 修改配置参数
[deploy]
mode = 'local'
cookie = 'domain'
data_dir = '/var/lib/sysagent'

[rag]
rag_service = 'http://127.0.0.1:9988'

[fastapi]
domain = '192.168.2.112'  # 修改为部署服务器 IP

[minio]
endpoint = '127.0.0.1:9000'
access_key = 'minioadmin'
secret_key = ''  # 部署脚本会自动生成
secure = false

[postgres]
host = '127.0.0.1'
port = 5432
user = 'euler_copilot'
password = ''  # 部署脚本会自动生成
database = 'euler_copilot'

[security]
half_key1 = ''  # 部署脚本会自动生成
half_key2 = ''
half_key3 = ''
jwt_key = ''

[extra]
sql_url = 'http://127.0.0.1:9015'
```

**说明：**

- `fastapi.domain`：必须修改为部署服务器的 IP 地址
- `security` 和 `postgres.password`、`minio.secret_key`：部署脚本会自动生成随机密码，无需手动填写

### 3. 执行一键部署

```bash
[root@localhost 5-resource]# witty-manager   #启动后直接输入 0 开启一键部署 等待部署完成，视网络情况好坏，大概在10~40分钟
一键部署菜单             
0) 自动部署
1) 手动部署
2) 重启服务
3) 卸载服务
4) 退出程序
请输入选项编号（0-4）: 

说明：0 自动部署将部署 sysagent 后端服务和相关依赖；1 手动部署支持分步骤执行环境检查、依赖安装和配置初始化。
一键部署完成后，需要调用下面的初始化agent来构建agent服务。
```

### 4. 初始化mcp agent

```bash
#创建mcp服务
witty-manager --a init config.json
#config.json 为copilot对应的mcp服务配置文件，参考如下，传入全路径
{
    "name": "systrace_mcp_server",
    "overview": "systrace 运维 mcp 服务",
    "description": "systrace 运维 mcp 服务",
    "mcpType": "sse",
    "author": "root",
    "config": {
        "env": {},
        "autoApprove": [],
        "disabled": false,
        "auto_install": false,
        "description": "",
        "timeout": 60,
        "url": "http://127.0.0.1:12145/sse"
    }
}
#说明：init 多次调用会删除之前注册的mcp 服务，重新注册
#创建一一对应的agent应用
witty-manager --a create config.json
#config.json 同上，是调用init之后，会在原始json里面添加serviceId字段标识mcp服务
{
    "name": "systrace_mcp_server",
    "overview": "systrace 运维 mcp 服务",
    "description": "systrace 运维 mcp 服务",
    "mcpType": "sse",
    "author": "root",
    "config": {
        "env": {},
        "autoApprove": [],
        "disabled": false,
        "auto_install": false,
        "description": "",
        "timeout": 60,
        "url": "http://127.0.0.1:12145/sse"
    },
    "serviceId":"p2qQke"
}
#创建多对一的agent应用
witty-manager --a comb config.json
#config.json是组合多个mcp创建agent
{
  "appType": "agent", #应用类型，不需要修改
  "icon": "", #图标
  "name": "agent_comb",  #创建应用的名字
  "description": "测试agent comb", #创建的应用的描述
  "dialogRounds": 3, #对话轮次，默认3
  "permission": {  #权限，不需要修改
    "visibility": "public",
    "authorizedUsers": []
  },
  "workflows": [], #工作流预留参数，不需要修改
  "mcpService": [
    {
      "id": "jFOWgw" #注册的mcp返回的serviceId
    },
    {
      "id": "4tA5TO" #注册的mcp返回的serviceId
    }
  ],
  "published": "True" #是否公开
}
#说明：如果是sse 或者 streamHttp格式 mcp server，在注册之前需要服务端先启动并且可用
```
